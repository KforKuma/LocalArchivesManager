# Public test corpus

The corpus lives in `examples/test_corpus/`. `manifest.json` is the source of
truth for identity, hashes, redistribution policy, and expected behavior;
`manifest.schema.json` defines its JSON Schema.

## Fixture matrix

| Case | Module/path under test | Expected route | Success criterion |
|---|---|---|---|
| `pdf_native_text` | PDF inspection and native extraction | `native_text_pdf`; no OCR required | Extracted text contains `Synthetic Signals`; clean metadata title is usable |
| `pdf_image_only` | Visual classification and OCR routing | image-only/scanned article candidate | Native extraction returns zero text and the page is eligible for OCR |
| `pdf_screenshot_wrapped` | Screenshot contamination classifier | screenshot-wrapped OCR route | Browser URL/chrome is not accepted as the article title |
| `pdf_contaminated_metadata` | PDF metadata quality filtering | reject metadata, retain page text | `Download PDF` metadata is rejected; native page title remains usable |
| `reference_refs1` | Reference segmentation and identifier extraction | explicit `reference_text=only` | Six candidates and six DOI candidates; no Documents rows |
| `reference_refs2` | 0.6.1 title-only regression path | explicit `only` plus `accept-title-only` target | Six one-line title candidates after that milestone; no guessed identifiers and no Documents rows |

`refs2` deliberately records the pre-title-only observation
`single_unconfirmed_group` in the manifest. The corpus infrastructure does not
change parser behavior; it makes the 0.6.1 acceptance target reproducible.

## Generate and verify included PDFs

```powershell
python scripts/generate_test_corpus.py
python -m pytest tests/test_test_corpus_061.py
```

The generator uses deterministic PDF settings. A successful regeneration is
byte-for-byte identical to the SHA-256 values in the manifest. The image-only
fixtures must have no extractable text layer. Visual QA can be repeated with:

```powershell
pdftoppm -png examples/test_corpus/included/native_text.pdf tmp/pdfs/native_text
```

Inspect the PNG for clipping, overlap, broken glyphs, and unreadable text.

## Optional external corpus

Only entries marked `redistribution=download_only` are processed:

```powershell
python scripts/fetch_test_corpus.py
```

The fetcher requires a direct HTTP(S) URL and a pinned SHA-256, writes only to
`downloaded/`, refuses to overwrite, verifies existing files, removes failed
partials, and applies a 100 MiB limit. It does not search shadow libraries,
authenticate to subscriptions, or attempt paywall circumvention.

No external paper is currently selected. Before adding one, a maintainer must
manually confirm the exact article/version, stable first-party or repository
URL, expected hash, license/terms, and the module behavior it uniquely tests.

## Provenance and copyright

All included PDFs are generated by LAM and contain invented text. The two text
fixtures contain citation/title strings only and are normalized public UTF-8/LF
copies. No article body, abstract corpus, user Catalogue, `.library_state`, or
topic note is distributed. Bibliographic facts are generally low-risk, but a
maintainer should still review unusual database export terms before expanding
the text corpus.
